Web RTC

What Is WebRTC?

WebRTC (Web Real-Time Communication) is a free and open-source project providing web browsers and mobile applications with real-time communication (RTC) via application programming interfaces (APIs).

Why Choose WebRTC?

WebRTC has a wide range of benefits for both the broadcaster and the receiving audience. Here are the top:

  • WebRTC is supported by most major web browsers, including Google Chrome for desktop and Android, Mozilla Firefox for desktop and Android, and Safari, and works on any operating system.
  • WebRTC does not require third-party components or plugins and is free open-source software.
  • WebRTC allowing direct peer-to-peer communication.

So how does all of this work?

There are four sequential steps that must be taken to establish a connection:

  • Signaling
  • Connecting
  • Securing
  • Communicating

Signaling

Signaling is used to initiate the call, allowing two independent WebRTC agents to start communicating.

Connecting

Once SDP offers are exchanged each peer will then make a series of requests to a STUN server which will generate a list of ICE candidates to use.

Securing

Security is achived through two more protocols: DTLS (Datagram Transport Layer Security) and SRTP (Secure Real-Time Transport Protocol).

Communicating

To make communication possible, WebRTC use two protocols: RTP (Real-time Transport Protocol) - to exchange media encrypted with SRTP, and SCTP (Stream Control Transmission Protocol) - to send and receive DataChannel messages encrypted with DTLS.

WebRTC - a collection of protocols

A good way to visualize it is a ‘WebRTC Agent’ is really just an orchestrator of many different protocols.

WebRTC API

There are three main categories of APIs that exist in WebRTC:

  • RTCPeerConnection
  • RTCDataChannel
  • Media Capture and Streams

What is WebRTC used for?

You can group WebRTC applications into 4 broad categories:

  • Conversational voice and video
  • Live streaming
  • Data transfer
  • Privacy
